How Helpdesk Priorities are Assigned

Very High: Problems that affect an entire library and stop all staff or clients from working

  • All Public workstations within a library cannot print
  • All Public workstations within a library cannot connect to the Library website
  • All staff within a library cannot logon to the network
  • All staff within a library cannot read their e-mail
  • All staff within a library cannot print to any of the shared network printers

High: Problems that prevent an individual staff member from working

  • A user cannot logon to the network
  • A user cannot print to any printer
  • A user cannot run one of the applications centrally supported by ITS (e.g. Office, Internet Explorer, PuTTY) or an application that accesses the central university administration systems (OASIS, EPIC, PISCES) and that application is necessary for them to work

Normal: Problems that prevent a user from performing some function but do not stop him/her from working altogether.

  • Restoring a file deleted from the H: drive or a share directory
  • All virus-related problems that cannot be resolved by the end-user on his/her own or with the help of his/her LAN Liaison
  • Fixing an application installed by network staff at the request of the user
  • All hardware problems that occur while the user has the option to log on to other workstations in his/her library
  • Disappearing desktop settings

Low: Problems that do not interfere with users performing primary job functions. Activities that can be scheduled in advance or dealt with through regular maintenance.

  • User cannot print from preferred printer
  • ID changes
  • Equipment moves
  • Request for software to be installed
  • Public workstations losing profiles
